## Installation

Install the TOON SDK for TypeScript/JavaScript:

<CodeGroup>

```bash npm
npm install @toon-format/toon
```

```bash pnpm
pnpm add @toon-format/toon
```

```bash yarn
yarn add @toon-format/toon
```

</CodeGroup>

**GitHub**: [github.com/toon-format/toon](https://github.com/toon-format/toon)
</Note>

## Your First Encoding

Encode JavaScript objects to TOON format using the `encode()` function:

```typescript
import { encode } from '@toon-format/toon'

const data = {
  users: [
    { id: 1, name: 'Alice', role: 'admin' },
    { id: 2, name: 'Bob', role: 'user' }
  ]
}

console.log(encode(data))
```

**Output**:

```yaml
users[2]{id,name,role}:
  1,Alice,admin
  2,Bob,user
```

<Note>
Notice how TOON declares the array length `[2]` and field names `{id,name,role}` once, then streams row values. This is the key to TOON's token efficiency.
</Note>

## Your First Decoding

Decode TOON strings back to JavaScript objects using the `decode()` function:

```typescript
import { decode } from '@toon-format/toon'

const toon = `users[2]{id,name,role}:
  1,Alice,admin
  2,Bob,user`

const result = decode(toon)
console.log(result)
```

**Output**:

```json
{
  "users": [
    { "id": 1, "name": "Alice", "role": "admin" },
    { "id": 2, "name": "Bob", "role": "user" }
  ]
}
```

<Note>
TOON provides **100% lossless, deterministic round-trips** between JSON and TOON. No data is lost in conversion.
</Note>

## Working with Real Data

Let's encode a more realistic dataset - employee records:

```typescript
import { encode } from '@toon-format/toon'

const employees = {
  company: 'Acme Corp',
  department: 'Engineering',
  employees: [
    { id: 101, name: 'Alice Johnson', salary: 95000, active: true },
    { id: 102, name: 'Bob Smith', salary: 87000, active: true },
    { id: 103, name: 'Carol White', salary: 92000, active: false }
  ]
}

console.log(encode(employees))
```

**Output**:

```yaml
company: Acme Corp
department: Engineering
employees[3]{id,name,salary,active}:
  101,Alice Johnson,95000,true
  102,Bob Smith,87000,true
  103,Carol White,92000,false
```

**Token Comparison**:
- JSON (formatted): ~180 tokens
- TOON: ~95 tokens
- **Savings**: ~47%

## Encoding Options

Customize encoding with options:

<CodeGroup>

```typescript Custom Indentation
import { encode } from '@toon-format/toon'

const data = { users: [{ id: 1, name: 'Alice' }] }

// Use 4 spaces for indentation (default is 2)
const toon = encode(data, { indent: 4 })
```

```typescript Tab Delimiters
import { encode } from '@toon-format/toon'

const data = { users: [{ id: 1, name: 'Alice' }] }

// Use tabs instead of commas (even more efficient!)
const toon = encode(data, { delimiter: '\t' })
// users[1]{id\tname}:
//   1\tAlice
```

```typescript Key Folding
import { encode } from '@toon-format/toon'

const data = {
  api: {
    config: {
      timeout: 5000
    }
  }
}

// Collapse single-key chains into dotted paths
const toon = encode(data, { keyFolding: 'safe' })
// api.config.timeout: 5000
```

</CodeGroup>

<Note>
See [Encode Options](/api/encode-options) for all available configuration options.
</Note>

## Streaming Large Datasets

For large datasets, use `encodeLines()` to stream output line by line without building the full string in memory:

```typescript
import { encodeLines } from '@toon-format/toon'

const largeData = await fetchThousandsOfRecords()

// Memory-efficient streaming
for (const line of encodeLines(largeData)) {
  process.stdout.write(`${line}\n`)
}
```

**When to use streaming**:
- Datasets with thousands of records
- Writing to files or HTTP responses
- Memory-constrained environments
- Real-time data processing

<Note>
For streaming decode, see [`decodeFromLines()`](/api/decode-from-lines), [`decodeStreamSync()`](/api/decode-stream-sync), and [`decodeStream()`](/api/decode-stream).
</Note>

## Using the CLI

For quick conversions without writing code, use the TOON CLI:

<Steps>
  <Step title="No installation needed">
    Use `npx` to run the CLI instantly:
    
    ```bash
    # Convert JSON to TOON
    npx @toon-format/cli input.json -o output.toon
    
    # Decode TOON to JSON
    npx @toon-format/cli data.toon -o output.json
    ```
  </Step>
  
  <Step title="Pipe from stdin">
    The CLI auto-detects format and supports pipes:
    
    ```bash
    # Pipe JSON directly
    echo '{"name": "Ada", "role": "dev"}' | npx @toon-format/cli
    
    # Chain with other tools
    curl https://api.example.com/data | npx @toon-format/cli
    ```
  </Step>
  
  <Step title="Analyze token savings">
    Use `--stats` to compare token counts:
    
    ```bash
    npx @toon-format/cli data.json --stats
    # TOON: 1,234 tokens
    # JSON: 2,456 tokens  
    # Savings: 49.8%
    ```
  </Step>
</Steps>

<Note>
See the [CLI Guide](/guides/cli) for all options and examples.
</Note>

## Working with LLMs

TOON works best when you **show the format** instead of describing it. The structure is self-documenting:

```typescript
import { encode } from '@toon-format/toon'
import Anthropic from '@anthropic-ai/sdk'

const client = new Anthropic()

const employees = [
  { id: 1, name: 'Alice', dept: 'Eng', salary: 95000 },
  { id: 2, name: 'Bob', dept: 'Sales', salary: 87000 },
  // ... 100 more employees
]

// Encode data as TOON
const toonData = encode({ employees })

const response = await client.messages.create({
  model: 'claude-4.5-sonnet',
  max_tokens: 1024,
  messages: [{
    role: 'user',
    content: `Here is employee data in TOON format:

\`\`\`toon
${toonData}
\`\`\`

How many employees in Engineering have salary > $90,000?`
  }]
})
```

<Note>
**Pro tip**: Wrap TOON in ` ```toon` code blocks for input. For generation tasks, show the expected header template and use tab delimiters for better efficiency.
</Note>

See the [LLM Integration Guide](/guides/llm-integration) for detailed strategies.

## Transforming Data with Replacer

Use the `replacer` function to filter or transform values during encoding:

<CodeGroup>

```typescript Remove Sensitive Fields
import { encode } from '@toon-format/toon'

const user = {
  name: 'Alice',
  email: 'alice@example.com',
  password: 'secret123',
  apiKey: 'sk-abc123'
}

const safe = encode(user, {
  replacer: (key, value) => {
    // Remove sensitive fields
    if (key === 'password' || key === 'apiKey') {
      return undefined
    }
    return value
  }
})

// Output:
// name: Alice
// email: alice@example.com
```

```typescript Transform Values
import { encode } from '@toon-format/toon'

const data = {
  status: 'active',
  category: 'premium',
  count: 42
}

const transformed = encode(data, {
  replacer: (key, value) => {
    // Uppercase all strings
    if (typeof value === 'string') {
      return value.toUpperCase()
    }
    return value
  }
})

// Output:
// status: ACTIVE
// category: PREMIUM
// count: 42
```

```typescript Add Metadata
import { encode } from '@toon-format/toon'

const data = { users: [{ id: 1, name: 'Alice' }] }

const withMeta = encode(data, {
  replacer: (key, value, path) => {
    // Add timestamp to root object
    if (path.length === 0 && typeof value === 'object' && value !== null) {
      return {
        _timestamp: Date.now(),
        _version: '1.0',
        ...value
      }
    }
    return value
  }
})
```

</CodeGroup>

<Note>
The `replacer` function is similar to `JSON.stringify`'s replacer but includes path tracking. See the [Replacer Function Guide](/advanced/replacer-function) for more examples.
</Note>
